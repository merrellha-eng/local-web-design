# LOCAL WEB DESIGN Webサイト制作指示書

## 0. 制作開始前に必ず確認すること

この指示書は、同時に共有する**参考LP画像**をWebサイトとして実装するための補足仕様書です。  
**デザインの見た目は参考LP画像を正とし、この指示書は画像だけでは判断できない文章・実装条件・公開環境の条件を指定するものとして扱ってください。**

制作開始前に、以下を必ず実施してください。

1. このMarkdownファイルを最初から最後まで確認する。
2. 同時に共有されている参考LP画像を確認する。
3. プロジェクト内の `images` フォルダに入っている画像素材をすべて確認する。
4. 参考LP画像に対し、`images` 内のどの画像を各箇所に使用するか整理する。
5. 既存ファイルがある場合は、構成と影響範囲を確認してから編集する。
6. 画像不足、文言の不一致、実装上の不明点がある場合は、推測で大きく変更せず、作業前に報告する。

### 参考LP画像と本指示書の扱い

- レイアウト、色、余白、写真の見せ方、文字サイズ感、装飾、全体の雰囲気は、**参考LP画像を優先**してください。
- 文言、リンク先、お問い合わせページの仕様、公開環境、使用素材の制限は、**本指示書を優先**してください。
- 本指示書にないデザイン要素を新しく追加して、参考LP画像の印象を変えないでください。
- 参考LP画像を一枚画像として貼り付けて再現することは禁止です。HTML / CSSで実装してください。

## 1. 制作概要

### サイト名

`LOCAL WEB DESIGN`

### サービス内容

地域密着型ホームページ制作サービス。  
主な対象は、ITやホームページ制作に詳しくない、個人経営の小さなお店・サロン・飲食店などのオーナーです。

### 作成するページ

| ファイル名 | 内容 |
|---|---|
| `index.html` | 参考LP画像を再現したトップページ |
| `contact.html` | お問い合わせフォームのダミーページ |
| `thanks.html` | ダミーフォーム送信ボタンクリック後に表示するページ |

## 2. 使用素材に関する必須条件

### 2-1. 画像

- Webページ内で使用する画像は、**必ず `images` フォルダ内にある画像のみ**を使用してください。
- 外部サイトから画像を取得しないでください。
- 新しい画像を生成しないでください。
- 参考LP画像は、デザイン確認用です。ページ全体の背景画像や一枚画像として使用しないでください。
- 必要な用途に合う画像が `images` 内にない場合は、無理に別画像を割り当てず、不足内容を報告してください。
- 画像の表示は、参考LP画像に近くなるよう `object-fit` や `object-position` を調整してください。
- 画像には適切な `alt` 属性を設定してください。

### 2-2. アイコン

アイコンは、Google Fonts Icons の **Material Symbols** から適切なものを選んで使用してください。

参照先：`https://fonts.google.com/icons`

- 基本スタイルは `Material Symbols Outlined` としてください。
- Font Awesomeなど、別のアイコンライブラリは使用しないでください。
- アイコン画像を新規作成しないでください。
- アイコンの種類・線の細さ・色は、参考LP画像に近いものを選択してください。

読み込み例：

```html
<link href="https://fonts.googleapis.com/css2?family=Material+Symbols+Outlined" rel="stylesheet">
```

```css
.material-symbols-outlined {
  font-variation-settings:
    'FILL' 0,
    'wght' 300,
    'GRAD' 0,
    'opsz' 24;
}
```

使用候補：

| 表示箇所 | 候補 |
|---|---|
| リーズナブルな価格 | `currency_yen` / `paid` |
| 素早く納品 | `schedule` |
| 追加料金なし | `verified` |
| お悩み：作り方が分からない | `person_search` / `help` |
| お悩み：価格 | `currency_yen` |
| お悩み：依頼先 | `signpost` |
| お悩み：不安 | `live_help` |
| お問い合わせ | `mail` |
| ヒアリング | `description` |
| コーディング | `code` |
| 確認・修正 | `edit` |
| 納品 | `redeem` |

## 3. GitHub Pages公開を前提とした実装条件

このサイトは、完成後にユーザーが手動でGitHubへ配置し、**GitHub Pagesで公開**します。  
Codexは公開作業を行わず、公開できる状態のファイル一式を制作してください。

- HTML / CSS / JavaScriptで動作する静的サイトとして制作してください。
- PHP、サーバーサイドNode.js、データベース、外部フォーム送信サービスなど、サーバー処理を必要とする実装は使用しないでください。
- **Formspreeは使用しないでください。**
- Bootstrap、Tailwind CSS、React等のフレームワークは使用せず、通常のHTML / CSS / JavaScriptで実装してください。
- 画像、CSS、JavaScript、ページ間リンクは、GitHub Pagesのリポジトリ配下公開でも壊れないよう**相対パス**で記述してください。
- 不要な依存関係やビルド処理は導入しないでください。
- `.nojekyll` を空ファイルとして作成してください。

## 4. トップページ `index.html`

### 4-1. 再現方針

トップページの構成・見た目は、共有された参考LP画像を確認し、できるだけ忠実に再現してください。  
画像から判断できる配色、セクションの高さ、配置、余白、罫線、ボタン形状、写真トリミング等を、この指示書で重複して細かく再定義する必要はありません。

### 4-2. 実装するセクション

参考LP画像に表示されている以下の範囲を実装してください。

### 4-3. 画像に写っていないセクションについて

ヘッダーナビゲーション内に「料金プラン」「よくある質問」等の項目が表示されていても、参考LP画像にそのセクション本体が掲載されておらず、別途内容が支給されていない場合は、**内容を推測して新規セクションを作成しないでください。**

対応方針：

- 参考LP画像上のナビゲーション表示自体は再現する。
- 対応するセクションの内容が未支給の場合、リンク先の扱いはTODOコメントを残した仮リンクとして扱う。
- 支給されているセクションへのリンクは、ページ内アンカーとして動作させる。
- 「お問い合わせ」は `./contact.html` にリンクする。

## 6. お問い合わせダミーページ `contact.html`

お問い合わせページは参考LP画像には含まれていないため、トップページの雰囲気を引き継いだ、シンプルで読みやすいデザインで作成してください。

### 6-1. このページの重要な扱い

- 現段階では、実際に問い合わせ内容を送信・保存・メール通知する機能は実装しません。
- フォームは表示確認用の**ダミーフォーム**として作成してください。
- Formspreeを含む外部送信サービスは使用しないでください。
- 入力内容をURLのクエリパラメータに含めて遷移させないでください。
- フォームのHTML付近に、運用開始前に送信機能の実装・差し替えが必要であることをコメントで残してください。

HTMLコメント例：

```html
<!-- TODO: 現在は表示確認用のダミーフォームです。運用開始前に実際のお問い合わせ送信方法を決定し、送信処理を実装してください。 -->
```

### 6-2. 表示内容

```text
CONTACT

お問い合わせ

ホームページ制作についてのご相談は、
下記フォームよりお気軽にお問い合わせください。
```

### 6-3. フォーム項目

| 項目 | 必須／任意 | input種別 |
|---|---|---|
| お名前 | 必須 | `text` |
| 店舗名・事業名 | 任意 | `text` |
| メールアドレス | 必須 | `email` |
| 電話番号 | 任意 | `tel` |
| ご相談内容 | 必須 | `textarea` |
| 個人情報の取り扱いへの同意 | 必須 | `checkbox` |

### 6-4. 送信ボタンの動作

- 必須項目が入力され、同意チェックが入っている状態で送信ボタンをクリックした場合、`./thanks.html` に遷移してください。
- 未入力の必須項目がある場合は、HTML標準の入力チェックを動作させ、`thanks.html` へ遷移させないでください。
- 遷移時に入力内容を送信・保存しないでください。
- URLに入力内容が表示されない実装にしてください。

実装方針例：

```html
<!-- TODO: 現在は表示確認用のダミーフォームです。運用開始前に送信処理を実装してください。 -->
<form id="contact-form">
  <!-- 入力項目を配置 -->
  <button type="submit">送信する</button>
</form>
```

```javascript
const contactForm = document.getElementById('contact-form');

if (contactForm) {
  contactForm.addEventListener('submit', (event) => {
    event.preventDefault();

    if (!contactForm.checkValidity()) {
      contactForm.reportValidity();
      return;
    }

    // 現段階では入力内容を送信せず、確認用の完了ページへ移動するだけ
    window.location.href = './thanks.html';
  });
}
```

### 6-5. 表示上の注意

- ページ上に「ダミーフォーム」や「現在送信できません」等の表示は、参考確認用サイトの見栄えを損なうため、通常表示では記載しないでください。
- ただし、ソースコード内のコメントと `README.md` には、実運用前に送信処理が必要であることを明記してください。
- 入力欄と送信ボタンはスマートフォンでも操作しやすくしてください。
- フォーム下部にトップページへ戻るリンクを設置してください。

## 8. 共通実装条件

### 8-1. HTML / CSS

- `header`、`nav`、`main`、`section`、`footer` など、意味のあるHTMLタグを使用してください。
- 見出しレベルを適切に設定してください。
- 画像には適切な `alt` を設定してください。
- CSSは `css/style.css` に整理してください。
- 参考LP画像の再現に必要な範囲で、Google Fontsを使用して構いません。
- リンクやボタンにはホバー状態とキーボードフォーカス状態を設定してください。

### 8-2. JavaScript

- JavaScriptは `js/main.js` に記述してください。
- 使用目的は、スマートフォン表示時のナビゲーション開閉、およびダミーフォームから `thanks.html` への遷移処理など、必要最小限にしてください。
- メニュー開閉ボタンには、`aria-expanded` などアクセシビリティに配慮した属性を設定してください。

### 8-3. レスポンシブ対応

参考LP画像はデスクトップ表示の基準として扱い、スマートフォンでは読みやすさと操作性を優先してレイアウトを再構成してください。

必須確認幅：

- デスクトップ：`1440px` 前後
- タブレット：`768px` 前後
- スマートフォン：`375px` 前後

確認事項：

- 横スクロールが発生しないこと
- ナビゲーションがスマートフォンで適切に開閉できること
- 文字が写真に埋もれず読めること
- 制作例カードや制作フローが狭い画面で無理なく並ぶこと
- フォームがスマートフォンで入力しやすいこと
- ダミーフォームで必須項目の入力チェック後、入力値を送信せずに `thanks.html` へ遷移すること
